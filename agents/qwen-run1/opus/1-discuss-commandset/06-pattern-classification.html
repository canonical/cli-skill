Recommendations

1. Add engines command

Priority: High | Type: Addition | Breaking: No

Per DE013: "use foobars as shorthand for listing all instances of a secondary object." Enables engine discovery.

Proposed: qwen36 engines → lists available engines with name, description, hardware match.

2. Add status command

Priority: High | Type: Addition | Breaking: No

Per DE013: "use shorthand status over show-status." Single command for system health.

Proposed: qwen36 status → engine, server state, component readiness.

3. Add unset command

Priority: Medium | Type: Addition | Breaking: No

Per DE013: standard config triple is get/set/unset. Enables restoring defaults.

Proposed: qwen36 unset <key> → removes user override.

4. Rename model-name config key

Priority: Medium | Type: Rename | Breaking: Yes (managed via deprecation)

High confusion risk with model key. Rename to api-model-name.

Deprecation: Keep alias for one cycle, warn, remove in next major.

5. Consider switch-engine

Priority: Low | Type: Rename (optional) | Breaking: Yes (managed)

use is non-standard in DE013 vocabulary. switch-engine is clearer but cost of change may exceed benefit.

Recommendation: Defer unless doing a broader grammar revision.

6. Add version command

Priority: Low | Type: Addition | Breaking: No

Per DE013 standard commands table. All comparison tools support this.
