Pattern Classification And Recommendations

The qwen36 CLI is a compact flat command set with one hidden debug namespace. Its strongest pattern is the config cluster get, set, and unset. Its weakest pattern is the interaction and engine surface, where noun-led webui and the split vocabulary of public use-engine versus hidden select-engine pull the grammar in different directions.

Pattern Classification

PropertyAssessment
Primary grouping patternflat top-level command set
Secondary grouping patternone hidden namespace: debug
Dominant grammarverb-led commands with a few noun exceptions
Strongest familyget / set / unset
Weakest familychat / webui plus use-engine vs hidden select-engine

Recommendations

  1. Fix the product-surface mismatch before renaming anything: either enable chat and webui in the shipped snap or remove them from public docs until they are enabled.
  2. Add engines as a public alias for list-engines. Per DE013 grammar, plural-noun shorthand is preferred for listing secondary objects.
  3. Introduce select-engine as the preferred public name and begin deprecating use-engine. Per DE013 grammar, command verbs should be clear and even.
  4. Introduce a verb-led launcher such as open-webui and deprecate webui. Per DE013 grammar, commands should be verbs.
  5. Add reset-engine or clear-engine as a reverse operation for engine choice.

Deprecation-sensitive recommendations above assume the required staged transition: add the new command in the next minor release, warn on the old form for at least one full release cycle, and remove the old name only in the next major release with exit code 2 and a replacement hint.
